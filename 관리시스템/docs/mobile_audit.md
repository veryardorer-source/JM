# JM OS — 모바일 대응 점검 & 수정 목록

> 점검일: 2026-06-18 · 대상: https://jm-interior.vercel.app (라이브 배포본)
> 목적: **PC 화면은 그대로 두고, 핸드폰에서 보기 편하게** 만들기 위한 작업 목록.
> ⚠️ 이 문서는 "준비"용. 실제 적용은 **앱 소스 코드를 작업 PC로 가져온 뒤** 진행한다.

## 현재 앱 구조 (라이브에서 확인)

- **기술스택 추정:** Next.js(App Router) + Tailwind. 화면은 클라이언트에서 그림(로딩 → 데이터 표시).
- **공통 레이아웃:** 왼쪽 **고정 세로 사이드바**(로고 + 메뉴 5개 + 하단 회사명) + 오른쪽 본문.
- **라우트:**
  - `/` 대시보드
  - `/notices` 공지사항
  - `/projects` 현장 관리 (목록=표)
  - `/projects/[id]` 현장 상세 (자료/공정 탭)
  - `/receipts` 영수증
  - `/withdrawals` 출금 요청

## 핵심 원칙

- 데스크톱 레이아웃은 **건드리지 않는다.** Tailwind 반응형 접두사로 "기본=모바일, `md:`/`lg:`=현재 PC 모습" 형태로만 얹는다.
- 화면 분기 기준: **`md` (768px)** 를 데스크톱/모바일 경계로 사용.
- 작업 단위가 작고 화면별로 독립적이라, **한 화면씩 고치고 바로 핸드폰에서 확인**하는 식으로 진행.

## 수정 목록 (우선순위 순)

### 1. 🔴 사이드바 → 모바일 햄버거 메뉴 (공통, 최우선)

- **문제:** 왼쪽 고정 사이드바가 좁은 폰 화면 가로 폭을 잡아먹음. 본문이 찌그러짐.
- **수정:**
  - 모바일(`< md`): 사이드바 숨김 + 상단에 **☰ 햄버거 버튼**. 누르면 사이드바가 슬라이드로 열림(오버레이).
  - 데스크톱(`md:`): 지금처럼 항상 펼침.
- **구현 메모:** 레이아웃 컴포넌트(사이드바 감싼 곳)에서 `hidden md:flex` + 모바일 토글 상태(useState) + 오버레이 `<div>`.
- **대안:** 메뉴가 5개라 **하단 탭바**(모바일 앱처럼)도 좋은 선택. 대표님 취향 확인 후 결정.

### 2. 🔴 대시보드 — 공정 타임라인 (`/`)

- **문제:** 6월~7월 날짜가 가로로 길게 펼쳐진 간트차트 → 폰에서 좌우 스크롤 지옥.
- **수정:**
  - 이미 있는 **`🗂 카드` 토글**을 **모바일에서 기본값**으로. (타임라인은 PC 기본)
  - 또는 타임라인 영역을 `overflow-x-auto`로 감싸 의도된 가로 스크롤로만 처리.
- **구현 메모:** 초기 뷰 상태를 화면폭에 따라 결정(`window.matchMedia('(min-width:768px)')`).

### 3. 🟡 대시보드 — 상단 통계 카드 (전체/진행중/완료)

- **문제:** 3개가 가로 한 줄 → 폰에서 답답.
- **수정:** `grid-cols-1 sm:grid-cols-3` (모바일 세로 1열 또는 `grid-cols-3`이면 `grid-cols-1`로).

### 4. 🟡 대시보드 — 직원별 업무 현황 카드

- **문제:** 카드 3개가 가로 1줄 → 폰에서 글자 깨짐.
- **수정:** `grid-cols-1 md:grid-cols-3` 로 모바일에서 세로 1열 쌓기.

### 5. 🔴 현장 관리 목록 — 표(테이블) (`/projects`)

- **문제:** 현장명/고객명/담당자/진행단계/시공일정/등록일 **6열 표** → 폰에서 가로 넘침.
- **수정:** 모바일에서는 **표 대신 카드 목록**으로. 한 현장 = 카드 1장(현장명 크게, 나머지는 라벨:값 형태로 세로 나열, 진행단계 배지).
- **구현 메모:** `<table className="hidden md:table">` + 모바일용 카드 `<div className="md:hidden">` 두 벌 렌더. (데이터는 같은 배열 재사용)

### 6. 🟡 현장 관리 — 검색/필터 줄

- **문제:** 검색창 + 필터칩 4개가 한 줄 → 폰에서 넘칠 수 있음.
- **수정:** `flex-wrap` + 검색창 `w-full`, 필터칩은 가로 스크롤 또는 줄바꿈.

### 7. 🟡 현장 상세 — 자료/공정 탭 (`/projects/[id]`)

- **자료 탭:** 사진 갤러리가 들어가면 `grid-cols-2 md:grid-cols-4` 같은 반응형 그리드로. `사진 선택`/`+자료 추가` 버튼은 모바일에서 가로폭 꽉 차거나 줄바꿈.
- **공정 탭:** (간트/작업목록 추정) 대시보드 타임라인과 동일 원리 — 모바일은 목록/카드형.

### 8. 🟢 공지사항 · 영수증 · 출금요청 (`/notices` `/receipts` `/withdrawals`)

- 현재 **빈 상태(empty state)**라 큰 문제 없음. 데이터가 쌓이면:
  - **영수증/출금요청:** 사진 카드 그리드 → `grid-cols-2 md:grid-cols-4`.
  - **공지사항:** 목록 카드 → 그대로 세로 쌓기면 OK. 상단 필터탭(전체/디자인팀/현장팀)만 `flex-wrap` 확인.
- 상단 우측 액션 버튼(+공지등록 / +사진업로드)이 모바일에서 제목과 겹치지 않게 `flex-col sm:flex-row` 점검.

### 9. 🟢 공통 — viewport 메타 / 글자 크기 / 터치 영역

- `app/layout.tsx`에 `viewport` 설정 있는지 확인(Next 15는 `export const viewport`). 없으면 추가.
- 버튼/링크 터치 영역 최소 44px 확보(`min-h-11` 등).
- 본문 좌우 여백이 모바일에서 너무 넓지 않게(`px-4 md:px-8`).

## 진행 절차 (소스 도착 후)

1. 작업 PC에서 앱 저장소 GitHub에 push → 이 PC로 `git clone`
2. `npm install` → `npm run dev` 로 로컬 실행, 핸드폰 크기로 확인하며 작업
3. 위 목록 **1번(사이드바)부터** 한 항목씩 적용 → 매번 모바일/PC 둘 다 확인
4. 완료되면 commit → push → Vercel 자동 배포(또는 `vercel` CLI)

## 확정된 결정 (2026-06-18 대표 확인)

- ✅ 모바일 메뉴: **☰ 햄버거 (좌상단, 슬라이드로 열림)** 방식으로 구현.
- ✅ 대시보드 공정표: 모바일에서는 **카드형으로 전환**(현장별 카드 목록). PC는 타임라인 유지.
