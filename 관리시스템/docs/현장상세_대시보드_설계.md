# 🏗️ 현장 상세 대시보드 — 화면 설계 (확정)

> 확정: 2026-06-22 (대표 승인 "이대로 진행"). 구현 위치: 앱 `/projects/[id]`
> ⚠️ 구현은 앱 소스 필요(현재 이 PC에 없음). 본 문서대로 구현한다.

## 목적
현장관리에서 **현장 하나를 열면, 그 현장의 모든 정보(공정·자료·비용·직원)가 한 화면에 대시보드처럼** 나오게 한다. 새 데이터는 만들지 않고 기존 테이블을 모아서 보여주는 "읽기 중심" 화면.

## 화면 구성 (위→아래)

### 1) 헤더
- 현장명 · 상태 뱃지(문의/실측/디자인/견적/계약/시공중/완료) · 주소 · 기간(start~end) · 담당자
- 데이터: `projects`(name, status, address, start_date, end_date), 담당자 = `project_assignments`/`projects.manager`

### 2) 요약 카드 4개
| 카드 | 값 | 출처 |
|------|-----|------|
| 공정 진행률 | 완료 공정 비율(%) | `schedules` 집계 |
| 현장 자료 | 자료 개수 | `project_files` count |
| 누적 원가 | 합계 금액 | `project_costs` sum |
| 출금 대기 | 승인 대기 건수 | `withdrawal_requests` (status=대기) |

### 3) 공정 현황 — **막대그래프(진행률) 방식 확정**
- 공정별(철거/목공/전기/타일/도장/필름/가구/청소 등) 한 줄씩, 진행률 막대 + 상태 라벨
- 상태색: 완료=초록, 진행중=주황(+%), 예정=회색, 지연=빨강
- 데이터: `schedules`(공정명/상태/진행률 또는 일정 기반 계산)

### 4) 현장 자료 — **사진+도면 한곳에 모아보기 확정**
- 썸네일 그리드(사진 우선) + 도면/문서는 아이콘, 우측 상단에 "사진 N · 도면 M", 마지막 칸 "+N 더보기"
- 데이터: `project_files`(종류/썸네일/이름)

### 5) 배정 직원 · 비용
- 배정 직원 칩(이름/이니셜): `project_assignments`
- 영수증(건수·합계) `receipts`, 출금 요청(대기 건수) `withdrawal_requests`, 누적 원가 `project_costs`

## 권한 처리 — **현장팀(field)은 돈 정보 숨김 확정**
- `role = field` → 요약카드의 "누적 원가", 5)의 영수증/출금/원가 **금액 숨김 또는 비표시**.
- `admin`, `designer`는 전체 표시. (정확한 designer 노출 범위는 구현 시 대표와 확인)

## 구현 메모
- 기존 현장 상세 페이지(자료/공정 탭 구조)를 **이 단일 대시보드 레이아웃으로 개편**.
- 시안 위젯과 동일한 구성. (모바일에선 카드가 세로로 쌓이도록 반응형 — `mobile_audit.md` 원칙 따름)
- 데이터는 모두 존재 → 추가 마이그레이션 불필요(단, `schedules`에 진행률 컬럼 없으면 일정 기반 계산 또는 컬럼 추가 검토).
