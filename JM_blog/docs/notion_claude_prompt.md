# Claude에게 전달할 프롬프트

아래 내용을 그대로 클로드에게 붙여넣으면 됩니다.

---

JM Notion 작업 이어서 해줘.

중요: 먼저 `docs/notion_claude_handoff.md`를 읽고, 거기에 적힌 최종 결정사항을 기준으로 작업해줘. 특히 아래 결정은 되돌리면 안 돼.

1. 전 직원 `진행중인 현장` DB에는 돈 관련 정보가 보이면 안 됨.
2. 경비/출금/재무/회계 relation을 전 직원 현장 DB에 양방향으로 다시 붙이면 안 됨.
3. 고객명/연락처/계약금액은 관리자 전용 DB에만 둬야 함.
4. 현장 사진은 DB 속성칸이나 현장 본문 직접 나열이 아니라 `시공전 사진첩 / 시공중 사진첩 / 완성 사진첩` 하위 페이지에서 관리.
5. 완공 현장은 별도 DB로 복사하지 말고 같은 현장 DB에서 `진행단계=완공` 필터로 분류.
6. 모바일 사용성이 최우선이므로 표(table)를 기본 보기로 두지 말고 갤러리/보드/리스트 중심으로 정리.

먼저 해야 할 일:

1. 새 토큰은 현재 PowerShell 창의 `NOTION_TOKEN` 환경변수로만 받아.
2. `notion_preflight_audit.py`를 먼저 실행해서 `CHECK` 항목이 있는지 확인해줘.
3. `notion_decorate.py`로 메인 대시보드에 직원 모바일 안내, 사진 삭제 안내, 폴더 바로가기를 적용해줘.
4. `notion_site_album_pages.py`로 현장 카드에 사진첩 하위 페이지가 있는지 확인해줘.
5. 새 `JM 업무 시스템`을 기준으로 작업하고, 옛 HUB는 백업으로만 취급해줘.
6. 아직 모바일 최적화가 덜 된 DB를 모바일용 보기로 바꿔줘.
   - SNS/블로그 발행
   - 자재/발주
   - A/S
   - 협력업체
   - 공지/매뉴얼
   - 직원 명부
   - 근태 기록
   - 매출/매입
   - 고객 CRM
   - 견적 현황
   - 공사 아카이브
7. 직원 초대 전 권한 기준을 다시 점검해줘.

로컬 파일 기준으로 토큰 제거, 환경변수화, `STRUCTURE.md` 정리는 완료됨.
잘못 올린 사진은 각 사진첩 하위 페이지에서 직원이 직접 삭제할 수 있게 안내해줘. 전 직원 현장 DB에 사진 파일 속성을 다시 만들지 마.

주의:

- Notion은 열 단위 권한이 안 되므로 민감정보를 전 직원 DB에 넣지 마.
- 뷰에서 숨기는 건 보안이 아니라 눈가림일 뿐이야.
- 전 직원 DB와 관리자 DB를 섞지 말고 권한 폴더 기준으로 분리해.
