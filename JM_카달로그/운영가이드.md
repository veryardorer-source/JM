---

## 1. 파일 구성

```
JM_카달로그/
  index.html      - 전체 카달로그 (모든 업종)
  office.html     - 사무실 고객용 카달로그
  public.html     - 관공서/학교 고객용 카달로그
  catalog.css     - 디자인 (공통)
  catalog.js      - 기능 (공통)
  images/
    projects/     - 프로젝트별 마감사진
      office_seokyung/   00.jpg, 01.jpg, ...
      beauty_barber/     00.jpg, 01.jpg, ...
      ...
    3d/           - 3D 렌더링 이미지
    process/      - 시공과정 사진
```

---

## 2. 포트폴리오 추가/수정 방법

### 새 프로젝트 추가하기

**1단계: 사진 준비**
- `images/projects/` 폴더 안에 새 폴더 생성
- 폴더 이름은 영문으로 (예: `office_newproject`)
- 사진을 `00.jpg`, `01.jpg`, `02.jpg`... 순서로 이름 변경 후 넣기
- 대표 사진(가장 잘 나온 것)을 `00.jpg`으로

**2단계: HTML에 항목 추가**
- 해당 카달로그 HTML 파일을 메모장이나 VS Code로 열기
- 추가하고 싶은 업종 페이지의 `port-grid` 안에 아래 코드 복사 후 수정:

```html
<div class="port-item" data-project="폴더이름" data-title="프로젝트명" data-location="위치 | 연도">
    <img src="images/projects/폴더이름/00.jpg" alt="">
    <div class="port-caption"><strong>프로젝트명</strong><span>위치 | 연도</span></div>
</div>
```

- 큰 이미지로 넣고 싶으면 `port-item` 뒤에 `port-large` 클래스 추가

### 사진 교체하기
- `images/projects/해당폴더/` 안의 jpg 파일을 같은 이름으로 교체하면 됨
- 예: 거울에 촬영자가 비치는 사진 → 다른 사진으로 `02.jpg` 교체

### 프로젝트 삭제하기
- HTML에서 해당 `port-item` 코드 블록 삭제
- (사진 폴더는 삭제 안 해도 됨)

## 3. 고객에게 보내는 방법

### 방법 1: 파일로 직접 전달 (가장 간단)
- `JM_카달로그` 폴더 전체를 **ZIP으로 압축**
- 카카오톡, 이메일 등으로 전송
- 고객이 압축 풀고 `office.html` (또는 `public.html`) 더블클릭하면 브라우저에서 바로 열림

### 방법 2: 웹호스팅에 올리기 (추천)
- 기존 홈페이지(jminterior.co.kr)의 서버에 올리면:
  - `jminterior.co.kr/catalog/` → 전체 카달로그
  - `jminterior.co.kr/catalog/office.html` → 사무실용
  - `jminterior.co.kr/catalog/public.html` → 관공서/학교용
- 고객에게 링크만 보내면 됨
- 호스팅 업체에 FTP로 폴더째 업로드하면 됨

### 방법 3: GitHub Pages (무료)
- GitHub 계정으로 무료 웹호스팅 가능
- 별도 서버 비용 없음

### 방법 4: PDF로 변환
- 브라우저에서 카달로그를 연 뒤 `Ctrl + P` → "PDF로 저장"
- 인쇄용으로도 활용 가능

## 4. 홈페이지와의 차이점

| 구분 | 홈페이지 (jminterior.co.kr) | 카달로그 |
|------|---------------------------|---------|
| **목적** | 회사 홍보, 검색 유입 | 특정 고객에게 직접 전달 |
| **대상** | 불특정 다수 | 상담 중인 특정 고객 |
| **내용** | 모든 정보를 한 곳에 | 고객 업종에 맞춘 포트폴리오 중심 |
| **맞춤화** | 한 가지 버전 | 사무실용/관공서용 등 고객별 버전 |
| **형태** | 웹사이트 (스크롤, 메뉴) | 브로슈어/룩북 (페이지 넘기기) |
| **전달** | URL 하나 | 링크 or 파일 or 인쇄물 |
| **활용** | "회사 홈페이지 보세요" | "저희 시공사례 카달로그입니다" |
| **인쇄** | 어려움 | Ctrl+P로 인쇄/PDF 가능 |
| **갱신** | 수시 업데이트 | 프로젝트 완료 시 추가 |

### 활용 시나리오
- 고객: "카달로그 있어요?"
  → 고객 업종에 맞는 버전 링크 또는 파일 전송
- 사무실 인테리어 상담 중
  → office.html 링크 전송 → 사무실 시공사례 집중 어필
- 관공서/학교 입찰 제안
  → public.html PDF 출력 → 제안서에 첨부

## 5. 연락처 수정

각 HTML 파일의 뒤표지(마지막 페이지)에서 아래 부분을 찾아 수정:
```html
<div class="back-label">ADDRESS</div>
<div class="back-value">경상남도 창원시</div>  ← 여기를 실제 주소로

<div class="back-label">TELEPHONE</div>
<div class="back-value">전화 문의</div>  ← 여기를 실제 전화번호로

<div class="back-label">E-MAIL</div>
<div class="back-value">이메일 문의</div>  ← 여기를 실제 이메일로
```

**3개 파일(index.html, office.html, public.html) 모두 동일하게 수정해야 합니다.**
